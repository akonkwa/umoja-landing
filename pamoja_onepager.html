PAMOJA
Agent-Based Social Navigation
Akonkwa Mubagwa (Harvard)  |  Oluwatise (MIT)  |  MIT AI Venture Studio 2026
Your network has always been a map - frozen in time. Pamoja turns it into a living system that remembers, reasons, and acts.
Problem / User

People attend high-value events, classes, and recurring community gatherings with no intelligent support for navigating who to meet, why they matter, or how one event connects to the next. Existing tools provide static profiles and attendance lists but do not accumulate social context over time. Valuable connections are left to luck.

Why Agentic
Each user is represented by a persistent Profile Agent with memory, goals, and callable workflows
Each event is represented by an Event Agent that understands attendees, themes, and context
Profile Agents query Event Agents, retrieve cross-event memory, rank who matters, and recommend concrete actions
After debrief, the Profile Agent updates memory so recommendations improve over time
MVP Loop
1. Organizer uploads an attendee list and event metadata
2. The system creates draft Profile Agents for attendees
3. User claims their agent and confirms: working on / looking for / want to meet
4. User asks: "Who should I meet at this event?"
5. Profile Agent queries Event Agent, retrieves cross-event memory, returns 3 ranked people with reasons
6. After the event, user debriefs who they met and what happened
7. Profile Agent updates memory, improving future recommendations
Success Metric
User receives 3 relevant recommendations in under 90 seconds. At least one recommendation is rated useful after the event debrief.
Why This Matters

The most important relationships in communities - co-founders, collaborators, mentors, early customers - emerge through repeated context, not one-off introductions. A system that retains and reasons over social context across events can make serendipity systematic.

Architecture
Why Us / Why Now

We are building for a setting we directly inhabit: recurring Harvard/MIT cohort-style communities where the same people repeatedly gather but lack intelligent social infrastructure. Function-calling LLMs, lightweight orchestration frameworks, and practical memory systems now make persistent but computationally efficient agents feasible.

Feasibility by Deadline
Build: Profile Agent creation from roster, claim/confirm flow, one Event Agent per event, one core query ("Who should I meet?"), ranked recommendations with explanations, post-event debrief with memory writeback
Won't build: full messaging platform, always-on autonomous agents, full graph explorer, generalized multi-agent simulations
Unique Moat
Persistent agent memory compounds in value over time
Cross-event intelligence is missing from all incumbents
Closed-cohort deployment solves cold start on day one
Every debrief strengthens future retrieval, ranking, and recommendations
Platform Vision

Pamoja is the agentic social layer for any recurring community - alumni networks, professional cohorts, conference circuits, housing co-ops, religious communities. A decentralized successor to static WhatsApp groups, where your agent carries your social memory across every group you belong to.
