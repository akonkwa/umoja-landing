PAMOJA

Agent-Based Social Navigation

Problem / User

People attend high-value events, classes, and recurring community gatherings with almost no intelligent support for navigating who to meet, why they matter, or how one event connects to the next.

Existing tools provide static profiles and attendance lists, but do not accumulate social context over time.

Core user question: “Who here should I meet, given who I am, what I want, and what has already happened across this community?”

Why This Matters

The most important relationships in communities, co-founders, collaborators, mentors, early customers, emerge through repeated context, not one-off introductions.

A system that retains and reasons over social context across events can make serendipity systematic.

Why Agentic

A normal app can store attendee lists, bios, and notes. It cannot reason across a person’s goals, event context, prior interactions, and evolving community history to decide what matters now and what should happen next.

  • Each user is a persistent dormant Profile Agent with memory, goals, and workflows.
  • Each event is an Event Agent that understands attendees, themes, and context.
  • Profile Agents query Event Agents, retrieve memory, rank who matters, and recommend actions.
  • After debrief, the Profile Agent updates memory so recommendations improve over time.
Feasibility + Metric

We are intentionally scoping the MVP to one workflow and one environment: repeated cohort/class events.

We will build: draft Profile Agent creation, claim flow, one Event Agent per event, one core query, ranked explanations, and post-event memory writeback.

We will not build: a messaging platform, full graph explorer, always-on agents, or generalized multi-agent simulations.

MVP success metric: a user can claim their agent and receive 3 relevant recommendations in under 90 seconds, and at least one recommendation is rated useful after debrief.

MVP Loop
  1. Organizer uploads attendees and event metadata.
  2. The system creates draft Profile Agents.
  3. User claims their agent and confirms three fields.
  4. User asks: “Who should I meet at this event?”
  5. Profile Agent returns 3 ranked people with reasons.
  6. Post-event debrief updates memory for the next event.
Why Us / Why Now

We are building for a setting we directly inhabit: recurring Harvard/MIT cohort-style communities where the same people repeatedly gather but lack intelligent social infrastructure.

Function-calling LLMs, lightweight orchestration frameworks, and practical memory systems now make persistent but computationally efficient agents feasible.

Unique Moat
  • Persistent agent memory compounds in value over time.
  • Cross-event intelligence is missing from incumbents.
  • Closed-cohort deployment solves cold start on day one.
  • Every debrief strengthens future retrieval, ranking, and recommendations.
Pamoja gives every person and every event an agent that can remember what matters, reason over evolving context, and help people build better relationships over time.
